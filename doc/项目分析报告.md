# 在线考试系统项目分析报告

## 一、项目概览

这是一个基于 **Spring Boot 2.x + Vue 2 + Ant Design Vue** 的前后端分离在线考试系统，采用 **JPA + MySQL + JWT** 技术栈。系统定义了三种角色（管理员/教师/学生）和三种题型（单选/多选/判断），核心数据模型见 [exam.sql](file:///persistent/home/wolf/spring-boot-online-exam/doc/sql/exam.sql)。

## 三、学生用户视角：考试体验分析

### 1. 答题流程体验

**优点：**
- 左侧导航按题型分组，已答题目有 👁 图标提示（[ExamDetail.vue#L28](file:///persistent/home/wolf/spring-boot-online-exam/frontend/src/views/list/ExamDetail.vue#L28)）
- 支持题目间切换且保留作答状态（`answersMap` 机制）
- 单选/多选/判断的交互组件选择合理

**严重体验问题：**

1. **无倒计时显示** — 页面写死"这里是倒计时"文字占位，学生无法感知剩余时间
2. **无交卷前确认** — [ExamDetail.vue#L191](file:///persistent/home/wolf/spring-boot-online-exam/frontend/src/views/list/ExamDetail.vue#L191) `finishExam()` 直接提交，误触即交卷，无法撤回
3. **无未答题目提醒** — 交卷时不检查是否有未作答题目
4. **无防误退出** — 刷新/关闭页面会丢失全部作答（`answersMap` 仅存内存）
5. **无草稿保存** — 网络中断即丢失答题进度
6. **题目无分页/滚动** — 一次只显示一题，大题量时导航效率低

### 2. 考后复查体验

**优点（[ExamRecordDetail.vue](file:///persistent/home/wolf/spring-boot-online-exam/frontend/src/views/list/ExamRecordDetail.vue)）：**
- ✅ 显示总分和参考时间
- ✅ 逐题查看对错（✓/✗ 图标）
- ✅ 答错题目显示正确答案
- ✅ 显示答案解析（`questionDescription`）

**不足：**
- ❌ 无法查看全班排名/分数分布
- ❌ 无法导出错题
- ❌ 无教师评语/反馈通道

### 3. 数据安全视角的隐患

- [application.yml#L7](file:///persistent/home/wolf/spring-boot-online-exam/backend/src/main/resources/application.yml#L7) **明文存储数据库密码** `anchor120.`
- [ExamController.java#L48](file:///persistent/home/wolf/spring-boot-online-exam/backend/src/main/java/lsgwr/exam/controller/ExamController.java#L48) 大量 `System.out.println` 打印敏感数据
- [ExamServiceImpl.java#L614](file:///persistent/home/wolf/spring-boot-online-exam/backend/src/main/java/lsgwr/exam/service/impl/ExamServiceImpl.java#L614) 打印用户答案到日志
- 无接口级权限控制，学生角色可调用 `/question/all`、`/exam/all` 等管理接口（仅前端路由权限，[LoginInterceptor](file:///persistent/home/wolf/spring-boot-online-exam/backend/src/main/java/lsgwr/exam/interceptor/LoginInterceptor.java) 仅校验登录态不校验角色）

## 四、综合评价

### 达成度总评

| 维度 | 评分 | 说明 |
|------|------|------|
| 基础架构搭建 | ★★★★☆ | 前后端分离、JWT、组件化做得规范 |
| 题库与组卷 | ★★★★☆ | CRUD 完整，分类（类型/难度/学科）设计合理 |
| 答题与判分 | ★★★☆☆ | 判分逻辑正确，但计时/有效期核心需求缺失 |
| 考试安全性与公平性 | ★☆☆☆☆ | 防作弊、时间约束、权限控制几乎为零 |
| 教学反馈闭环 | ★★☆☆☆ | 仅有基础复查，无统计分析、无错题本 |
| 生产可用性 | ★★☆☆☆ | 明文密码、System.out 日志、无角色级 API 鉴权 |

### 结论

该项目作为一个 **Spring Boot + Vue 的教学演示项目** 是合格的：架构清晰、代码结构规范、覆盖了在线考试的核心数据流（题库→组卷→答题→判分→复查）。

但作为一个 **可实际投入教育场景的在线考试产品**，存在以下致命短板：

1. **计时功能名存实亡** — 这是考试系统区别于普通问卷的核心特征，目前完全缺失
2. **考试有效期未生效** — 教师无法控制考试开放窗口
3. **零防作弊机制** — 所有学生同卷同序，无任何作弊检测
4. **后端无角色鉴权** — 前端权限可被绕过，学生可调管理接口
5. **无教学数据分析** — 教师无法掌握班级整体情况，丧失"考试作为教学反馈工具"的核心价值

项目 README 的 Todo 列表显示作者已意识到大部分短板（如第7项"老师能考到所有学生的成绩"、第8项"数据分析功能"），但均未实现。建议优先级：**计时强制约束 > 后端角色鉴权 > 考试有效期校验 > 题目随机化 > 成绩统计分析**。
